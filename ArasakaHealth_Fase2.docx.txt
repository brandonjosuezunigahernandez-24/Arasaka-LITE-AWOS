1. Convenciones de la API
1.1 Nomenclatura de Rutas
* Prefijo global: /api/v1/ — todas las rutas comienzan con este prefijo.
* Sustantivos en plural y kebab-case: /api/v1/patients, /api/v1/appointments.
* Recursos anidados sólo cuando la relación es obligatoria: /api/v1/patients/{id}.
* Sin verbos en las rutas — la acción la define el método HTTP (GET, POST, PUT, PATCH, DELETE).
* IDs de recursos como parámetros de ruta: /api/v1/appointments/{id}.


1.2 Métodos HTTP y su Semántica
Método
	Acción
	Uso en este proyecto
	Código éxito
	GET
	Leer recurso(s)
	Listar pacientes, listar citas, ver perfil
	200 OK
	POST
	Crear recurso
	Registro, login, crear cita
	201 Created
	PUT
	Reemplazar recurso
	Actualizar perfil completo del paciente
	200 OK
	PATCH
	Modificar parcialmente
	Cambiar status de una cita (confirmed/cancelled)
	200 OK
	DELETE
	Eliminar recurso
	Eliminar paciente, cancelar cita (soft delete)
	204 No Content
	

1.3 Estructura Estándar de Respuestas JSON
Respuesta exitosa (2xx)
{
 "success": true,
 "message": "Descripción del resultado",
 "data": { ... }   // objeto o arreglo según el endpoint
}


Respuesta de error (4xx / 5xx)
{
 "success": false,
 "message": "Descripción del error",
 "errors": {
   "campo": ["Mensaje de validación específico"]
 }
}


Respuesta vacía exitosa (204)
Sin cuerpo — solo el código de estado HTTP 204 No Content.


1.4 Códigos HTTP Utilizados
Código
	Nombre
	Cuándo se usa
	200
	OK
	Consulta, actualización o acción exitosa con cuerpo de respuesta.
	201
	Created
	Recurso creado exitosamente (registro de usuario, creación de cita).
	204
	No Content
	Eliminación o cancelación exitosa sin cuerpo de respuesta.
	400
	Bad Request
	Petición malformada o con datos incoherentes (no es problema de validación de campo).
	401
	Unauthorized
	Token ausente, inválido o expirado.
	403
	Forbidden
	Token válido pero el rol del usuario no tiene permiso sobre este recurso.
	404
	Not Found
	El recurso solicitado (paciente, cita) no existe en la BD.
	409
	Conflict
	Conflicto de estado: cita duplicada en el mismo horario.
	422
	Unprocessable Entity
	Validación de campos fallida (email duplicado, fecha pasada, etc.).
	500
	Internal Server Error
	Error inesperado del servidor — nunca se expone el stack trace al cliente.
	

1.5 Cabeceras Requeridas
Cabecera
	Valor
	Cuándo
	Content-Type
	application/json
	En toda petición con body (POST, PUT, PATCH).
	Accept
	application/json
	En toda petición — fuerza respuesta JSON.
	Authorization
	Bearer {token}
	En todos los endpoints protegidos (excepto /login y /register).
	

1.6 Formato de Datos
* Fechas y horas: ISO 8601 — ejemplo: 2026-04-15T10:30:00Z
* Zona horaria: UTC en la API. El frontend convierte a zona local.
* Decimales: punto como separador decimal (68.5 no 68,5).
* IDs: enteros auto-increment (bigint unsigned) en MySQL.
* Enums: strings en minúsculas — role: 'doctor' | 'patient', status: 'pending' | 'confirmed' | 'cancelled'.
* Contraseñas: nunca se retornan en ninguna respuesta.

POST

	Crea un nuevo usuario con rol doctor o patient. Si el rol es patient, también crea su PatientProfile vacío.

	Pública — no requiere token.
	Cabeceras
	Content-Type: application/json   |   Accept: application/json
	Códigos HTTP
	201 · 422
	Request body:
{
 "name":     "Brandon Zuñiga",
 "email":    "brandon@arasakahealth.mx",
 "password": "Secure123!",
 "password_confirmation": "Secure123!",
 "role":     "patient"
}
Respuesta exitosa:
HTTP 201 Created
{
 "success": true,
 "message": "Usuario registrado correctamente.",
 "data": {
   "token": "2|aB3kLm9zXqRt...",
   "user": {
     "id":    1,
     "name":  "Brandon Zuñiga",
     "email": "brandon@arasakahealth.mx",
     "role":  "patient",
     "created_at": "2026-03-26T18:00:00Z"

}

 "success": false,

   "password": ["La contraseña debe tener al menos 8 caracteres."],
   "role":     ["El rol debe ser doctor o patient."]
 }
}

	201 · 401 · 403 · 409 · 422

 "success": true,

{

   "scheduled_at": ["La fecha debe ser posterior a la fecha actual."],

PATCH
	/api/v1/appointments/{id}

 "status": "confirmed"

// Valores válidos:

 "success": true,

 "data": {
   "id":           1,
   "scheduled_at": "2026-04-15T10:30:00Z",

 "success": false,

 "errors": {}

3. Diseño de Base de Datos (MySQL 8.0)
3.1 Motor y Configuración
* Motor: MySQL 8.0 — charset utf8mb4, collation utf8mb4_unicode_ci.
* Conexión desde Laravel: driver mysql en config/database.php, variables en .env.
* Tablas gestionadas exclusivamente por migraciones de Laravel (Eloquent).
* IDs: bigint unsigned auto-increment (convención de Laravel).
* Timestamps: created_at y updated_at en todas las tablas (gestionados por Eloquent).

3.6 Código de Migraciones
patient_profiles
Schema::create('patient_profiles', function (Blueprint $table) {
   $table->id();
   $table->foreignId('user_id')->constrained()->cascadeOnDelete()->unique();
   $table->string('curp', 18)->nullable()->unique();
   $table->string('blood_type', 3)->nullable();
   $table->decimal('weight_kg', 5, 2)->nullable();
   $table->smallInteger('height_cm')->nullable();
   $table->string('phone', 20)->nullable();
   $table->timestamps();
});


appointments
Schema::create('appointments', function (Blueprint $table) {
   $table->id();
   $table->foreignId('patient_id')->constrained('users')->cascadeOnDelete();
   $table->foreignId('doctor_id')->constrained('users')->cascadeOnDelete();
   $table->dateTime('scheduled_at');
   $table->enum('status', ['pending','confirmed','cancelled'])->default('pending');
   $table->text('notes')->nullable();
   $table->timestamps();
   $table->index(['patient_id', 'scheduled_at']);
});

"require": {
   "php":                "^8.3",
   "laravel/framework":  "^11.0",

}


4.2 Comandos de ejecución
Comando
	Descripción
	composer install
	Instala todas las dependencias del proyecto.
	cp .env.example .env
	Crea el archivo de variables de entorno.
	php artisan key:generate
	Genera la APP_KEY del proyecto.
	php artisan migrate
	Ejecuta todas las migraciones en MySQL.
	php artisan migrate:fresh --seed
	Resetea la BD y ejecuta seeders (desarrollo).
	php artisan serve
	Inicia el servidor en http://localhost:8000
